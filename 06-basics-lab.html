Position

Position determines how an element is positioned in the document flow

  • Does the element stay still when you scroll the page?
  • Can elements overlap?
Position: Static

                    p { position: static; }
  • Default value
  • Elements render in order
  • Elements render as they appear in document flow
Position: Absolute

                    p { position: absolute; }
  • Elements position relative to its closest non-static ancestor element
  • Positions based on entire document if all elements above are static
  • Removes element from document flow
  • Useful for
    • Precise Positioning
    • Difficult alignment
Position: Fixed

                    p { position: fixed; }
  • Elements position based on the browser window (viewport)
  • Wont move when scrolling
  • Removes element from document flow
  • Useful for
    • Creating headers, side-nav, back to top buttons
    • Precise Positioning
    • Difficult alignment
Position: Relative

            p { position: relative; }
  • Element position based on its normal position in document flow
  • Adding left: 20px; will push element 20px left of its normal position in the document flow
Position: Sticky

                p { position: sticky; }
  • Element position based on user's scroll position
  • Combination of relative and fixed positions
  • Relative until a scroll offset is reached, then it "sticks" in place using fixed positioning
  • Not supported in IE/Edge 15
Position: Inherit

                p { position: inherit; }
  • Inherits the position property from parent element
