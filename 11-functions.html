Functions

What are functions?
  • A reusable section of code that has a purpose and a name
  • Sometimes called methods or subroutines
  • Make code much more readable
  • Basically when done right, it makes code beautiful
What do they do?

They can perform any code!

  • Calculations
  • Animations
  • Change CSS
  • Communicate with a server
  • Anything!
Defining functions

            // A Function Declaration 
            function sayHello() {    
                console.log( "Hello" ); 
            } 
            
            // A Function Expression
            var sayHi = function() {    
                console.log( "Hi" ); 
            };
How are functions called

Unlike normal code, functions will not run until they are called.

Return Functions

Return Values
  • Sometimes we want a method to return a value
  • Useful for splitting large algorithms into smaller bite sized pieces
Return Values

        function multiply (num1, num2) {    
            return num1 * num2; // return is a reserved word
        } 
    
        multiply(1, 5);     // Calls the function but value is lost
        var result = multiply(1, 5); // result is set to 5
Return Values
  • When return is hit, the function will exit
  • It is always the last line executed of a return function

        function multiply (num1, num2) {    
            return num1 * num2;
            console.log('value sent!') // Wont execute.
        }

Software engineering principles

What are the principles?
  • DRY - Dont Repeat Yourself
  • YAGNI - You Ain't Gonna Need It
  • KISS - Keep It Super Simple
  • Hint - Talk about these in job interviews! It would score you bonus points with me
DRY

Principle aimed at reducing repetition

  • "Every piece of knowledge should have a single, unambiguous representation in the system"
  • Allows for easier modifications to code because it should only happen in one place
  • Violations of this principle is called WET code (waste everyone's time, write everything twice)
Identifying violations
  • If you copy and pasted a block of code, its probably violating the DRY principle
  • If you do a modification to the system and it involves replacing the same parts of code in multiple files. Its probably a violation
  • If a modification is visible in one part of the system but not in another (when expected). There has probably been a violation

CODE ALONG: DRY functions

YAGNI

Basically, dont write code until its planned, budgeted and required

  • Writing code that will never be used sucks!
  • Writing code and finding out it's not needed sucks!
  • Writing code and finding out a client doesn't want it sucks!!
  • Wasting 8 months on a data migration process for corporate to scrap the idea.. SUCKS!!
KISS

Make functions easy to read by reducing their complexity

  • Everyone function should do one thing and one thing well!
  • Makes naming things easier if nothing else
  • Makes code more testable (something you will have to worry about in the future)
Identifying violations
  • A function has too many lines (I try for under 30)
  • Its hard to name a function, that generally means its doing more than one thing and needs a complicated name
  • It takes more than 5 minutes to understand what its doing (this one is pretty subjective)

Anonymous functions

Anonymous Functions
  • A function without a name
  • Can be used to execute code as soon as its loaded
  • Can also be used to encapsulate a block of code so it is not visible to other blocks of code
Anonymous Function

            // Named function
            function hello() {
                console.log("hello");
            }
            hello();

            // Anonymous function assigned to the hello variable
            var hello = function() {
                console.log("hello");
            }
            hello();
IIFE
  • Immediately-invoked function expression
  • A very important anonymous method
  • Runs as soon as it is defined
  • Variables within the expression can not be accessed from outside
IIFE Uses

            (function () { 
                var name = "Evan Wallace";
            })();

            console.log(name); // Throws an error
IIFE Uses

        (function () { 
            doPageLoadLogic();
            callServer();
            updateStylesViaJS();
        })();
