Why animations?

Why Animations?
  • We evolved to excel at noticing movement
  • Using movement and visual cues, we are able to understand complex actions
  • Web animations leverage that understanding
  • Helps make a deeper connection between user and website
Some technical reasons to use animations
  • Loading page > white page when loading large files
  • Spinner > nothing when processing a request on a server
  • Bouncy elements to highlight errors or call to actions
  • So many more!

CSS Properties
  • @keyframes
  • animation-name

@keyframes

        @keyframes slidein {
            from {
                margin-left: 100%;
                width: 300%;
            }
            
            to {
                margin-left: 0%;
                width: 100%;
            }
        }
  • Controls the steps between animation frames
  • Can use % or to-from
@keyframes cont

    @keyframes identifier {
        0% { top: 0; left: 0; }
        30% { top: 50px; }
        68% { left: 50px; }
        100% { top: 100px; left: 100%; }
    }
animation-name

    @keyframes identifier {
        0% { top: 0; left: 0; }
        100% { top: 100px; left: 100%; }
    }

    #animated-thing {
        animation-name: identifier;
    }
  • Uses the keyframe name to know which animation to use
animation-duration

    #animated-thing {
        animation-name: cool-animation;
        animation-duration: 3s;
    }
  • Time it takes animation to animate one cycle

animation-delay

    #animated-thing {
        animation-name: cool-animation;
        animation-delay: 3s;
    }
  • Allows you to delay the start of an animation.
  • Useful for orchestrating a few animations on a page

Code along: animation-delay

animation-iteration-count

    #animated-thing {
        animation-name: cool-animation;
        animation-iteration-count: infinite;
    }
  • Number of times animation will cycle
  • Default value is 1

Code along: animation-iteration-count

animation

    #animated-thing {
        animation: slidein 3s linear 1s infinite running;
    }
  • Shorthand for all the animation properties
  • Can sometimes be confusing which thing represents what

Hosting

Types of Hosting
  • Shared
  • Virtual Private Server
  • Dedicated
  • Cloud
Shared
  • One server with many websites
  • Server resources are shared (RAM, Hard drive etc)
  • Usually the cheapest option
  • Easy to set up
Virtual Private Server
  • One server set up to be many virtual servers
  • Shared physical resources but more control/security
  • Good balance of price and control
Dedicated Server
  • One server with one website on it
  • Very performant
  • Highest level of control and security
  • Most expensive
  • Tends to be overkill of simple websites
  • You often need to manage the security yourself
Cloud Hosting
  • Lots of servers working together
  • Best reliability and performance
  • Lots of options for hosting
  • Very scalable
How do you decide?
