Git/github

What is version control?
  • System to record changes to a file or set of files
  • Super useful to help fix any errors
  • Great tool for pointing the figure at someone when something goes wrong
What is git?
  • Version control system
  • Tiny program that is lightning fast
  • Industry standard for storing and collaborating with code
  • Free and open source
What is github?
  • Service that uses git
  • Similar to Dropbox or Google drive, but for code
  • Free for average users
  • Amazing tools for reaching out to the developer community

Repository
  • Data structure used to store files
  • Synonym for project
  • A project has 1 repository
  • The repository usually lives on a server (like github)
  • A team of developers have a copy of this repository and make changes there

Branch
  • Branches are copies of your code that you can change in isolation
  • Isolation is important. Developers create branches for each new feature, enhancement or bug fix they make to a repository
  • This allows other developers the change to review/collaborate
  • Repositories always start with a master branch
  • Repositories can have many branches (it gets messy when there are too many though)

Commit
  • A commit is a bunch of work (code changes)
  • Git requires you to add a commit, this helps you know what work was done
  • Commits also mark who created them, which is a great tool to blame people when something goes wrong
  • A branch usually has a few commits

Merging Branches
  • All roads lead to rome, All branches (should) merge to master
  • Once a block of work is finished you want to add those changes to master
  • This allows the rest of your team to have access to your changes
  • Once the branch is merged, its good practice to delete the feature branch
Master Branch
  • Represents the latest code in a project
  • Most projects dont allow developers to commit directly to master
  • Master is just the default that git uses. You can name your default branch whatever you want.
