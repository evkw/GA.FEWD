Why Responsive?

First computers to use the net

Pretty much all of these early monitors had a resolution of 800x600

So there was only one resolution to design for

Monitors got thinner but fit more pixels

By 2002 1024 x 768 became the most common display and many websites were re-designed to take advantage of the new space

In 2007 iphone changed everything
  • Had a resolution of 480 x 320
  • Started the race for smart phones
Smart phone resolutions are silly
Now add tablets in the mix
Its just crazy!!

All the while, computer monitors keep evolving and you could start browsing the net on your TV

Became un-manageable
  • Having to re-design a website for each resolution is not possible
  • Whole design disciplines were created to figure the best layout for the best device
  • CSS3 added tools in 2012 to help make life easier for everyone
Responsive design
  • Responsive designs respond to their environment
  • Became a cost-effective alternative to mobile development
  • Future proofed your websites from resolution craziness

Layout types

Fixed Layout
  • You have already made sites with this layout type
  • Has a wrapper element with a fixed width
  • Components inside it either have % or a fixed width/height
  • No matter what screen resolution, users experience the same width
Fixed Layout
Fixed Layout Pros/Cons
Pro Con
Easier to use and customize in terms of design May create excessive white space on larger resolutions
Less hassle with images, forms, video and other content that are fixed-width Smaller screen resolutions may require a horizontal scroll bar
No need for min-width or max-width Seamless textures, patterns and image continuation are needed to accommodate those with larger resolutions
Fluid layout
  • Also known as liquid layout
  • Majority of components use % widths to adjust to changes in screen
  • Width change with browser window size
  • Box-sizing: border-box helps a lot
  • Liquid fills the parent container width
Fluid Layout
Fluid Layout Pros/Cons

Amount of extra white space is similar between all browsers and screen resolutions Content with set widths may be difficult to work with
Can eliminate horizontal scroll bars in smaller screen resolutions On large resolutions, a lack of content may create excess white space

Viewports
  • User's visible area of a web page
  • Varies with device
  • Scales down a large desktop-first design to fit on a small mobile screen
  • Was introduced as a quick fix to display on multiple device. By itself, its not perfect!

No viewport set vs Viewport set

    Reposnive Design

width=device-width - Use device width to determine how site will look

initial-scale=1.0- Dont zoom in by default

A couple of rules
  • Dont use large fixed width elements. It will create a horizontal scroll bar
  • Dont let content rely on a particular viewport width
  • Use media queries to apply different styles for small and large screens
Fluid Sizing
  • Sizing elements to adapt to the browser
  • Dont hard code size values (we have done a lot of this in previous classes)
  • Particularly important for images
Fluid Sizing - cont.
                        
    img {
        width: 500px;
        max-width: 100%;
        height: auto;
    }

You can make an image scale by using a combination of any width, max-width and height: auto - Example

Units
  • em
  • rem
  • percentages
  • vw
  • vh
  • vmin
  • vmax
em
  • Relative to the font-size of the element
  • 2em means 2x the current font size
  • Play-around
rem
  • Also font based
  • Operates off the font-size of the root element (html)
  • Play-around
vh/vw
  • Based on the viewport
  • 1vh/vw = 1% of the view height/width
  • Play-around
vmin/vmax
  • Also based on the viewport
  • 1vmin = 1% of smallest value between height and width
  • 1vmax = 1% of largest value between height and width
  • Play-around
Media queries
  • Filters that can be applied to CSS styles
  • Uses conditions to determine whether the CSS is applied or not
                    
    @media ( condition ) { ... }
Media queries cont.
                    
    @media (min-width: 700px) { 
        ... 
    } 

    @media (min-width: 700px) and (orientation: landscape) { 
        ... 
    } 

    @media (min-width: 700px), handheld and (orientation: landscape) { 
        ... 
    }

    @media only screen and (color) {
        ... 
    }

    @media (not (tv)){ 
        ... 
    }
